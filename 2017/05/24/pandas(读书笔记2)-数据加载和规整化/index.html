合并数据集

merge(left, right)

参数解析
  1. how:
    • inner(默认)：交集
    • outer：并集
    • left：左连接
    • right：右连接
  2. on=None： 默认已left和right列名的交集为连接键
  3. left_on=None： left的连接键
  4. right_on=None： right的连接键
  5. left_index=False： 将left的列索引作为连接键
  6. right_index=False：将right的列索引作为链接建
  7. sort=False： 根据连接键合并后的数据进行排序
  8. suffixes=(‘_x’, ‘_y’)：有重叠列名时，追加在重叠列名的末尾
  9. copy=True：是否将数据复制到结果数据结构中
  10. indicator=False：
示例

concat(objs)

参数解析
  1. objs : 参与链接的对象
  2. axis : 链接的轴向，默认为0
  3. join : inner/outer,是并集还是交集运算
  4. join_axes : 指明用于其他n-1条轴的索引，不执行并集/交集索引
  5. verify_integrity : 检查结果对象上的重复情况
  6. keys : 与连接对象有关的值，用于形成连接轴向上的层次化索引
  7. levels : 指定用作层次化索引各级别上的索引
  8. names : 用于创建分层级别的名称
  9. ignore_index : 不保留连接轴上的索引，产生一组新索引
  10. copy : 是否拷贝数据

离散化

cut(x, bins)

  1. x: 数据
  2. bins：分割数据
  3. right: True：右中括号(默认)， False：左中括号
  4. labels=None: 划分的分组
  5. retbins=False: 是否返回bins
  6. precision=3: 数据返回精度
  7. include_lowest=False: 第一个间隔左边是否是[号

qcut(x, q)：根据样本分位数对数据进行面元划分

  1. x: 数据
  2. q: 数字或数组
  3. retbins=False: 是否返回bins
  4. precision=3: 数据返回精度
  5. labels: 用于存储返回的bins
