openCV + Python 安装

发表于 2016-10-14   |   分类于 Python ， OpenCV

mac 机器：

mac机器python环境比较规范，安装比较顺利，一次就安装成功

必须软件：

  1. python
  2. numpy
  3. cmake

如果没有，提前安装软件，安装参考 CentOS6.5上配置Python2.7+OpenCV2.4.7

步骤

  1. 点击下载opencv源代码, 并解压，进入到解压目录
  2. 新建release目录（mkdir release）

  3. 执行以下命令

    cmake -D CMAKE_BUILD_TYPE=RELEASE -D CMAKE_INSTALL_PREFIX=/usr/local -D BUILD_PYTHON_SUPPORT=ON -D BUILD_EXAMPLES=ON ..

    ==注意有没有报错信息，有错误一定要仔细查看==

  4. 执行：make -j8，此时注意release/lib下面是否有cv.so文件，如果没有，表示有问题，查看下具体的问题
  5. 执行：make install
  6. 查看python的site-package下面是否包含cv.py和cv2.so文件（在python的每个site-package目录下都查找下，目录用sys.path去查看）
  7. 测试，在终端输入python打开命令行客户端，输入import cv2，如果没报错就证明安装成功了

Centos6.5安装

服务器机器python被安装的比较乱，直接按照mac的cmake命令总是出错，最终采用指定python环境的方式去装

命令

cmake -D PYTHON_EXECUTABLE=/usr/bin/python \
-D PYTHON_PACKAGES_PATH=/usr/lib64/python2.6/site-packages/ \
-D INSTALL_PYTHON_EXAMPLES=ON \
-D PYTHON_INCLUDE_DIR=/usr/include/python2.6 \
-D PYTHON_LIBRARY=/usr/lib64/libpython2.6.so \
..

==看到上面命令就可以知道python的路径有多乱了==

错误

cmake -D PYTHON_EXECUTABLE=/usr/local/bin/python2 \
-D PYTHON_PACKAGES_PATH=/usr/local/python/python2/lib/python2.7/site-packages/ \
-D INSTALL_PYTHON_EXAMPLES=ON \
-D PYTHON_INCLUDE_DIR=/usr/local/python/python2/include/python2.7 \
-D PYTHON_LIBRARY=/usr/local/python/python2/lib/libpython2.7.a \
..

2.6.6的library是so文件，2.7的确实libpython2.7.a,然后就有问题了……

ompile with -fPIC
